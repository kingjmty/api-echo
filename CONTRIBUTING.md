# Contributing

Thank you for your interest in contributing! Here's how to get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/api-echo.git`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Run tests: `npm test`
6. Commit: `git commit -m "feat: your feature description"`
7. Push: `git push origin feature/your-feature`
8. Open a Pull Request

## Code Style

- Use ES6+ JavaScript
- Add JSDoc comments for public functions
- Keep functions small and focused
- Write descriptive variable names

## Commit Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation changes
- `test:` adding tests
- `chore:` maintenance

## Reporting Bugs

Use the Bug Report issue template. Include reproduction steps and environment info.

## Feature Requests

Use the Feature Request issue template. Describe the problem you're solving.

## Pull Request Process

1. Update README.md with any API changes
2. Update CHANGELOG.md with your changes under `[Unreleased]`
3. Ensure CI passes
4. Request a review from a maintainer
